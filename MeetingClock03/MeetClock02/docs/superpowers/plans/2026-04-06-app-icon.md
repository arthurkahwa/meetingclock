# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce and ship the approved burning-banknote app icon for MeetClock across iOS (light/dark/tinted) and macOS, plus wire the AccentColor.

**Architecture:** All artwork is built in Figma at 1024×1024 and exported as PNGs. Three iOS variants (light, dark, tinted) go into the `AppIcon.appiconset`. macOS needs ten PNGs (unique sizes 16→1024px). The tinted slot is left empty — iOS falls back to the light variant and applies the system tint (T-A decision). AccentColor is set to `#E8621A` in code.

**Tech Stack:** Figma (vector artwork), Xcode asset catalog (JSON + PNGs), SwiftUI AccentColor

## Task 1: Set up the Figma file

**Files:** Figma (external tool)

- [ ] **Step 1: Create Figma file named `MeetClock Icon`**

  Create three pages: `Light`, `Dark`, `Review`.

- [ ] **Step 2: On the Light page, create a 1024×1024 frame**

  Name it `AppIcon-Light`. Enable `Clip content`. Do NOT add a corner radius — Xcode/iOS masks the superellipse automatically.

- [ ] **Step 3: Add the background**

  Fill the frame with a radial gradient:
  - Center: `#1A1210`
  - Edge (100%): `#0D0908`
  - Gradient type: Radial, centered, radius = 100% of frame

- [ ] **Step 4: Verify background reads as near-black with a warm undertone**

  At 100% zoom the center should look slightly warmer than pure black. If it reads as flat black, increase the center color to `#221815`.

## Task 2: Draw the banknote body

**Files:** Figma `AppIcon-Light` frame

- [ ] **Step 1: Create a rectangle for the bill**

  Dimensions: 614×256px (roughly a 2.4:1 aspect ratio, like a real note scaled down).  
  Position: centered in the 1024×1024 frame.  
  Rotation: 8° clockwise.  
  Fill: `#4A5240` (desaturated olive green).  
  Corner radius: 6px.

- [ ] **Step 2: Add a thin dark border**

  Add an inner stroke: 3px, color `#2A2E24` (darker than fill). This defines the bill edge without a separate rectangle.

- [ ] **Step 3: Add the face area (inner panel)**

  Create a rectangle 540×190px, centered within the bill, same rotation.  
  Fill: `#3E4535` (slightly darker than bill body, simulating the engraved interior).  
  No border.

- [ ] **Step 4: Add the corner numbers**

  Create a text element: typeface `SF Mono` or `Courier New`, size ~28px, color `#8A9478`, content `$`.  
  Duplicate to all four corners of the inner panel (approx 20px from each corner edge), rotated 8° to match the bill.

- [ ] **Step 5: Verify bill proportions at 60px preview**

  Zoom out so the frame appears ~60px wide. The bill rectangle should read as a distinct light shape against the dark background. If it doesn't contrast enough, lighten the bill fill to `#5C6650`.

## Task 3: Draw the clock easter egg

**Files:** Figma `AppIcon-Light` frame, inside the bill face area

- [ ] **Step 1: Create the clock circle**

  Ellipse: 120×120px.  
  Position: center-left of the inner face panel (left third of the bill).  
  Fill: `#E8E0C8` (warm cream, simulating engraving paper).  
  No border.

- [ ] **Step 2: Add clock hands**

  Two lines from the center of the ellipse:
  - **Hour hand**: length 32px, angle pointing to 10 (300°), stroke `#2A2E24`, width 5px, round cap.
  - **Minute hand**: length 44px, angle pointing to 2 (60°), stroke `#2A2E24`, width 3px, round cap.

  Together these approximate 10:10.

- [ ] **Step 3: Add a center dot**

  Ellipse: 10×10px, centered on clock center, fill `#2A2E24`.

- [ ] **Step 4: Verify at 200px preview**

  The clock face should read as a recognizable analog clock. If the hands are hard to see, increase stroke widths by 1px each.

- [ ] **Step 5: Group the clock elements**

  Select all clock elements, group as `clock-easter-egg`. This makes it easy to hide/show when testing small-size legibility.

## Task 4: Draw the burn and char zone

**Files:** Figma `AppIcon-Light` frame

- [ ] **Step 1: Create the char mask**

  Duplicate the bill rectangle (from Task 2, Step 1). Name it `char-mask`.  
  Move it to the top of the bill layer stack.

  Using the Pen tool, edit the top edge of `char-mask` to be jagged and irregular — not a straight line. The bottom edge stays straight. This shape will define where the bill "ends" and char begins. The irregular top edge should vary by ±20–40px from a notional straight line across the top 40% of the bill.

- [ ] **Step 2: Apply the char gradient**

  Fill `char-mask` with a linear gradient (bottom to top):
  - 0% (bottom of mask): `#4A5240` at 0% opacity — blends into clean bill
  - 50%: `#2A1A0A` (dark char brown) at 90% opacity
  - 100% (top): `#1A0A00` (near-black ash) at 100% opacity

- [ ] **Step 3: Add the curled corner**

  On the top-right corner of the bill, create a small bezier shape that suggests a page curl — a thin crescent shadow on the bill surface (~80px wide, ~30px tall), fill `#1A0D00`, placed just below the char zone. This is subtle; if it looks like a mistake at 100px preview, delete it.

## Task 5: Draw the flame

**Files:** Figma `AppIcon-Light` frame

- [ ] **Step 1: Draw the outer flame shape**

  Using the Pen tool, draw an organic flame shape rising from the top of the bill. Two or three asymmetric tongues — do NOT mirror-symmetrize. The base of the flame starts at the top edge of the bill (aligned with the char zone). The tallest tongue reaches ~870px from the bottom of the frame (leaving ~154px of clear space above it before the frame edge).

  Fill: `#C0390B` (deep burnt red).  
  Set layer blend mode to `Normal`.

- [ ] **Step 2: Draw the mid flame shape**

  Duplicate the outer flame, scale it down ~25% from the center-bottom anchor point (narrower and slightly shorter).  
  Fill: `#E8621A` (ember orange).

- [ ] **Step 3: Draw the inner core shape**

  Duplicate mid flame, scale down another 25%.  
  Fill: linear gradient bottom-to-top, `#F5A623` → `#FFF0A0`.

- [ ] **Step 4: Add inner glow to the core**

  Add a Drop Shadow to the inner core layer:  
  X: 0, Y: 0, Blur: 24, Spread: 0, Color: `#F5A623` at 60% opacity.  
  This creates a soft halo effect simulating heat glow.

- [ ] **Step 5: Verify flame silhouette at 60px**

  The flame should read as a recognizable flame shape even at small sizes. The outer shape is the most important — it defines the silhouette. If the three layers merge into a blob, increase the size difference between layers (outer > mid > inner should each be clearly smaller).

## Task 6: Draw the embers

**Files:** Figma `AppIcon-Light` frame

- [ ] **Step 1: Create ember template**

  Ellipse: 8×8px. Fill: `#F5A623`. Name it `ember`.

- [ ] **Step 2: Duplicate and scatter 5 embers**

  Place embers in the region above and to the right of the flame. Suggested positions (approximate, from top-left of frame):
  - Ember 1: (680, 130), opacity 75%, size 8×8
  - Ember 2: (740, 95), opacity 55%, size 6×6
  - Ember 3: (620, 110), opacity 65%, size 5×5
  - Ember 4: (700, 72), opacity 40%, size 4×4
  - Ember 5: (760, 140), opacity 50%, size 7×7

  The positions should feel like particles drifting up-right from the flame tip, not arranged in a grid.

- [ ] **Step 3: Add a soft blur to the two smallest embers**

  Select Ember 4 and Ember 5. Add Layer Blur: 1.5px. This suggests motion and atmospheric scatter.

- [ ] **Step 4: Group all embers**

  Group as `embers`. Verify no ember touches the frame edge.

## Task 7: Create the dark variant

**Files:** Figma `AppIcon-Dark` frame (on the Dark page)

- [ ] **Step 1: Duplicate the entire Light frame to the Dark page**

  Copy `AppIcon-Light`, paste to the Dark page, rename to `AppIcon-Dark`.

- [ ] **Step 2: Update the background gradient**

  Change the radial gradient:
  - Center: `#0F0B0A`
  - Edge: `#080504`

- [ ] **Step 3: Desaturate the bill body**

  Change bill fill from `#4A5240` to `#3E4636` (slightly more grey, less green saturation). Inner panel from `#3E4535` to `#353C2E`.

- [ ] **Step 4: Flame and embers stay identical**

  Do not change any flame or ember colors. They're already optimized for dark backgrounds.

- [ ] **Step 5: Review side by side on the Review page**

  Create a new frame on the Review page, paste both icons at 256×256px side by side. Verify:
  - Dark variant reads as visibly different from light (not just marginally darker)
  - Flame colors read consistently across both
  - Bill is legible against background in both

## Task 11: Device testing

**Files:** None — visual QA only

- [ ] **Step 1: Test light icon on device**

  Install on a physical iPhone. In light mode, the home screen icon should show the full-color burning bill on the charcoal background. Verify the flame reads clearly and the bill silhouette is distinct.

- [ ] **Step 2: Test dark icon on device**

  Switch iPhone to dark mode. The icon should visibly shift — darker background, slightly more muted bill. Flame colors should remain warm and consistent.

- [ ] **Step 3: Test tinted behavior**

  Go to Settings → Accessibility → Display & Text Size → Color Filters. Enable Greyscale. The icon's layered structure (bill shape, flame tongues, embers) should remain readable as distinct layers. If bill and flame merge into the same grey value, note it for a potential T-B revision.

- [ ] **Step 4: Test macOS icon at 16×16**

  On macOS, check the app icon in the Dock at minimum size and in Launchpad. Only the flame silhouette and bill rectangle need to be legible at this size. The clock easter egg will not be visible — this is expected per spec.

- [ ] **Step 5: Test spotlight / share sheet thumbnails**

  On iPhone, long-press the app icon. The share sheet uses a small (~60px) version. Verify the icon reads correctly at that size.
